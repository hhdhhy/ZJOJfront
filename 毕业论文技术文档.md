# ZJOJ 在线判题系统 - 前端项目技术说明

## 项目概述

ZJOJ 是一个基于 Vue 3 的在线编程判题系统前端应用,为教练和学生提供题目管理、代码提交评测、AI 知识问答、班级管理和学情分析等功能。

**核心技术栈**:
- Vue 3 (Composition API)
- Vite 7 (构建工具)
- Element Plus (UI 组件库)
- Pinia (状态管理)
- Vue Router 4 (路由管理)
- Axios (HTTP 客户端)
- Docker (容器化部署)

## 二、核心功能模块详解

### 2.1 用户认证与权限控制

#### 2.1.1 JWT Token 认证流程

系统使用 JWT (JSON Web Token) 进行身份认证,Token 存储在 localStorage 中,每次请求通过 Authorization 头自动携带。

**完整认证流程**:
1. 用户输入用户名密码登录
2. 后端验证后返回 JWT Token 和用户信息(包含 role 字段)
3. 前端将 Token 和用户信息保存到 localStorage 和 Pinia Store
4. Axios 请求拦截器自动在 headers 中添加 `Authorization: jwt <token>`
5. Token 过期或无效时(401),自动清除登录状态并跳转登录页

**关键实现** (`src/stores/auth.js`):
```javascript
export const useAuthStore = defineStore('auth', () => {
    let _user = ref({})
    let _token = ref("")

    function setUserToken(user, token) {
        _user.value = user
        _token.value = token
        localStorage.setItem(USER_KEY, JSON.stringify(user))
        localStorage.setItem(TOKEN_KEY, token)
    }

    function clearUserToken() {
        _user.value = {}
        _token.value = ""
        localStorage.removeItem(USER_KEY)
        localStorage.removeItem(TOKEN_KEY)
    }

    return { setUserToken, clearUserToken, user, token, is_login }
})
```

### 3.1 路由管理与导航守卫

使用 Vue Router 实现前端路由,采用 hash 模式(`/#/`)避免服务器配置问题。

**路由配置** (`src/router/index.js`):
```javascript
const routes = [
  {
    path: '/',
    name: 'login',
    component: Login
  },
  {
    path: '/frame',
    name: 'frame',
    component: Frame,
    meta: { requiresAuth: true }  // 需要认证
  }
]

**路由守卫实现**:
```javascript
router.beforeEach((to, from, next) => {
  const authStore = useAuthStore()
  
  // 需要认证的页面
  if (to.meta.requiresAuth && !authStore.is_login) {
    next('/login')  // 未登录跳转登录页
  } 
  // 已登录用户访问登录页,跳转到首页
  else if (to.name === 'login' && authStore.is_login) {
    next('/frame')
  }
  else {
    next()  // 正常放行
  }
})
```

**路由元信息**:
```javascript

  path: '/frame',
  name: 'frame',
  component: Frame,

### 3.2 HTTP 请求封装与拦截器

基于 Axios 封装统一的 HTTP 客户端,处理认证、错误等通用逻辑。

**基础配置** (`src/api/http.js`):
```javascript
import axios from 'axios'

const http = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL,
  timeout: 180000,  // 3分钟超时,适应长时间评测
  headers: {
    'Content-Type': 'application/json'
  }
})

export default http
```

**认证拦截器** (`src/api/authHttp.js`):
```javascript
authHttp.interceptors.request.use((config) => {
  const authStore = useAuthStore()
  const token = authStore.token
  if (token) {
    // JWT认证格式: Authorization: jwt <token>
    config.headers.Authorization = `jwt ${token}`
  }
  return config
})
```

**重要**: JWT 认证格式是 `jwt <token>` 而不是标准的 `Bearer <token>`,这是后端自定义的要求。

**响应拦截器**:
```javascript
authHttp.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error.response?.status === 401) {
      // Token失效,清除登录状态并跳转登录页
      const authStore = useAuthStore()
      authStore.clearUserToken()
      ElMessage.error('登录已过期,请重新登录')
      window.location.href = '#/login'
    }
    return Promise.reject(error)
  }
)
```

#### 3.3.3 Pinia Store(全局状态)

适合全局共享的状态,如用户认证信息。

**认证 Store** (`src/stores/auth.js`):
```javascript
export const useAuthStore = defineStore('auth', () => {
  const _user = ref({})
  const _token = ref("")
  
  // 计算属性: 从 localStorage 恢复
  const user = computed(() => {
    if (!_user.value.username) {
      const stored = localStorage.getItem('OJ_USER_KEY')
      if (stored) _user.value = JSON.parse(stored)
    }
    return _user.value
  })
  
  const is_login = computed(() => {
    return Object.keys(user.value).length > 0 && token.value
  })
  
  return { user, token, is_login, setUserToken, clearUserToken }
})
```

**使用方式**:
```javascript
const authStore = useAuthStore()

// 读取
console.log(authStore.user.username)
console.log(authStore.is_login)

// 修改
authStore.setUserToken(userData, token)
authStore.clearUserToken()
```

**持久化**: 数据同时保存在内存(ref)和 localStorage,刷新页面后自动恢复。

### 3.5 错误处理与用户反馈

#### 3.5.1 统一错误提示

使用 Element Plus 的 ElMessage 组件统一展示错误信息。

**最佳实践**:
```javascript
try {
  await createClass(data)
  ElMessage.success('创建成功')
} catch (err) {
  // 优先显示后端返回的具体错误
  let errorMsg = '创建失败'
  if (err.response?.data?.message) {
    errorMsg = err.response.data.message
  } else if (err.response?.data?.detail) {
    errorMsg = err.response.data.detail
  }
  ElMessage.error(errorMsg)
  console.error(err)  // 开发时保留控制台日志
}
```

**错误信息优先级**:
1. 后端返回的 `message` 字段(最具体)
2. 后端返回的 `detail` 字段
3. 默认错误提示

**示例**:
- 班级名重复: "该班级已存在"
- 权限不足: "只有教练可以创建班级"
- 网络错误: "创建失败"

#### 3.5.2 加载状态管理

异步操作时显示 loading 状态,避免用户重复点击。

**实现方式**:
```javascript
const loading = ref(false)

const fetchData = async () => {
  loading.value = true
  try {
    const res = await apiCall()
    // 处理数据
  } finally {
    loading.value = false  // 无论成功失败都关闭 loading
  }
}
```

**UI 展示**:
```vue
<el-table v-loading="loading" :data="list">
  <!-- 表格内容 -->
</el-table>

<el-button :loading="submitting" @click="handleSubmit">
  提交
</el-button>
```

**防重复提交**:
```javascript
const sending = ref(false)

const sendMessage = async () => {
  if (sending.value) return  // 正在发送,忽略
  sending.value = true
  try {
    await sendRequest()
  } finally {
    sending.value = false  // 恢复状态
  }
}
```

### 3.6 样式管理与主题定制

#### 3.6.1 Scoped CSS

使用 Vue 的 scoped 样式,避免组件间样式冲突。

```vue
<style scoped>
.class-name {
  color: #409eff;
}
</style>
```

编译后变成:
```css
.class-name[data-v-xxxxx] {
  color: #409eff;
}
```

每个组件有唯一的 `data-v` 属性,确保样式隔离。

#### 3.6.2 Element Plus 样式覆盖

使用 `:deep()` 选择器覆盖 Element Plus 组件的内部样式。

```vue
<style scoped>
.input-field :deep(.el-input__wrapper) {
  padding: 6px 8px;
  border-radius: 4px;
  font-size: 12px;
}

/* 需要 !important 确保优先级 */
.custom-tag :deep(.el-tag) {
  font-size: 12px !important;
}
</style>
```

**注意事项**:
- `:deep()` 可以穿透 scoped 作用域
- 某些情况需要 `!important` 提高优先级
- 修改后需要硬刷新浏览器(Ctrl+F5)清除缓存

## 四、部署与运维

### 4.1 Docker 多阶段构建

采用多阶段构建策略,最终镜像只包含静态文件,体积极小。

**Dockerfile**:
```dockerfile
# 第一阶段: 构建
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci  # 安装所有依赖(包括 devDependencies)
COPY . .
RUN npm run build  # 生成 dist 目录

# 第二阶段: 运行
FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

**关键点**:
- 使用 `node:20-alpine` 基础镜像(体积小)
- `npm ci` 比 `npm install` 更快更可靠
- 最终镜像基于 `nginx:alpine`,只有几 MB
- 多阶段构建避免将 node_modules 打包进最终镜像

### 4.2 Nginx 反向代理配置

Nginx 同时处理静态资源和 API 转发。

**配置文件**:
```nginx
server {
    listen 80;
    server_name localhost;
    
    # 静态资源
    location / {
        root /usr/share/nginx/html;
        index index.html;
        try_files $uri $uri/ /index.html;  # SPA 路由 fallback
    }
    
    # API 转发到后端
    location /api/ {
        proxy_pass http://101.35.233.33:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        
        # 超时设置(适应长时间评测)
        proxy_connect_timeout 180s;
        proxy_send_timeout 180s;
        proxy_read_timeout 180s;
    }
}
```

**关键配置说明**:
- `try_files $uri $uri/ /index.html`: SPA 路由必需,否则刷新页面会 404
- `proxy_pass`: 将所有 `/api/` 请求转发到后端 Django 服务
- `timeout 180s`: 代码评测可能需要较长时间,设置 3 分钟超时

### 4.3 Docker Compose 编排

前端和后端独立部署,通过 Docker network 通信。

**前端专用配置** (`docker-compose.frontend.yml`):
```yaml
version: '3.8'
services:
  zjoj-frontend:
    build: .
    container_name: zjoj-frontend
    restart: always
    ports:
      - "80:80"
    networks:
      - zjoj-network

networks:
  zjoj-network:
    external: true  # 使用后端创建的网络
```

**部署命令**:
```bash
# 首次部署
docker compose -f docker-compose.frontend.yml up -d --build

# 更新代码后
git pull
docker compose -f docker-compose.frontend.yml up -d --build
```

**优势**:
- 前端和后端解耦,可以独立更新
- 使用外部网络,前后端容器可以互相访问
- `restart: always` 确保容器崩溃后自动重启

## 五、常见问题与解决方案

### 5.1 前后端权限字段不一致导致 403 错误

**问题现象**: 教练点击"创建班级"返回 403 Forbidden

**根本原因**: 
- 前端使用 `is_staff` 字段判断权限
- 后端使用 `role` 字段(role=2 为教练)
- 两个字段不同步,导致前端认为有权限,后端拒绝请求

**解决方案**: 统一使用 `role` 字段

修改前:
```javascript
const isCoach = computed(() => authStore.user?.is_staff)
```

修改后:
```javascript
const isCoachOrAdmin = computed(() => authStore.user?.role >= 2)
```

**涉及文件**(共 6 个):
- `src/components/HeaderNav.vue` - 身份标签显示
- `src/components/Sidebar.vue` - 菜单权限控制
- `src/views/main/frame.vue` - 面包屑导航
- `src/views/main/ClassList.vue` - 创建班级按钮
- `src/views/main/KnowledgeBase.vue` - 知识库管理权限
- `src/views/main/StudentReport.vue` - 报告类型切换

**经验教训**: 前后端权限判断必须使用同一字段,最好在 API 文档中明确约定。

### 5.2 班级列表数据解析错误导致列表为空

**问题现象**: 创建班级成功,但列表页面显示空白

**根本原因**: 后端返回格式与前端期望不一致

后端返回:
```json
{
  "code": 200,
  "message": "获取成功",
  "data": [
    {"id": 1, "name": "班级1"}
  ]
}
```

前端期望(DRF 分页格式):
```json
{
  "results": [...],
  "count": 10
}
```

**解决方案**: 修改前端数据解析逻辑

修改前:
```javascript
classList.value = res.data.results || []
total.value = res.data.count || 0
```

修改后:
```javascript
classList.value = res.data.data || []
total.value = classList.value.length
```

**影响范围**: 所有使用自定义 API 格式的接口都需要适配,包括:
- 班级列表
- 班级成员列表
- 知识库列表
- 学情报告

**经验教训**: 前后端联调时要仔细检查返回数据格式,不要假设使用标准 DRF 分页。

### 5.3 JWT 认证 Header 格式错误

**问题现象**: 所有需要认证的接口返回 401 Unauthorized

**根本原因**: JWT 认证 Header 格式不匹配

前端发送:
```
Authorization: Bearer eyJhbGci...
```

后端期望:
```
Authorization: jwt eyJhbGci...
```

**解决方案**: 修改 Axios 拦截器

```javascript
// 错误写法
config.headers.Authorization = `Bearer ${token}`

// 正确写法
config.headers.Authorization = `jwt ${token}`
```

**后端验证逻辑** (`MYJWT/authentication.py`):
```python
keyword = 'jwt'  # 自定义关键字

if parts[0].lower() != self.keyword.lower():
    return None  # 格式不匹配,跳过认证
```

**经验教训**: 仔细阅读后端认证代码或 API 文档,不要想当然使用标准的 Bearer 格式。

### 5.4 创建班级后整个页面刷新

**问题现象**: 创建班级成功后,整个页面刷新,回到初始状态

**根本原因**: 使用了 `window.location.reload()`

```javascript
// 不好的做法
if (currentPage.value === 'classes') {
  window.location.reload()  // 刷新整个页面
}
```

**解决方案**: 使用组件 ref 调用刷新方法

**子组件暴露方法**:
```javascript
// ClassList.vue
defineExpose({
  refresh: fetchClassList
})
```

**父组件调用**:
```javascript
// frame.vue
const classListRef = ref(null)

const handleCreateClass = async () => {
  await createClass(data)
  ElMessage.success('创建成功')
  
  // 只刷新列表数据,不刷新整个页面
  if (currentPage.value === 'classes' && classListRef.value) {
    classListRef.value.refresh()
  }
}
```

**优势对比**:

| 方式 | 用户体验 | 性能 | 状态保持 |
|------|---------|------|----------|
| window.location.reload() | 差,白屏闪烁 | 低,重新加载所有资源 | 丢失 |
| 组件 ref 刷新 | 好,无感知 | 高,只请求必要数据 | 保持 |

**经验教训**: 尽量避免整页刷新,使用局部更新提升用户体验。

### 5.5 班级成员 API 缺少 GET 方法

**问题**: ClassMemberView只有POST和DELETE方法,缺少GET方法获取成员列表,返回405错误。

**解决方案**: 在后端添加GET方法,返回成员列表数据。
